ABOUT THE SYSTEM

TR-PHISH is a production-ready, real-time Security Operations Center (SOC) platform designed to audit URLs, parse emails, decode QR codes, and run computer vision brand spoofing analyses to classify targets into Legitimate, Suspicious, or Phishing.

It utilizes a trained Random Forest Classifier combined with live domain intelligence socket feeds (WHOIS, DNS, and SSL verification).

LOCAL INSTALLATION

To run the website and its dynamic Python backend locally:

1. Initialize Virtual Environment

python -m venv venv
venv\Scripts\activate

2. Install Requirements

pip install -r requirements.txt

3. Train the Model

python ml/training/train.py

4. Run Flask Server

python -m app

Navigate to http://localhost:5000 and log in with:

  • Username: admin
  • Password: admin123

DOCKER CONTAINERIZATION

You can package and run the application inside a container:

# Build Image
docker build -t ai-phishing-detector .

# Run Container
docker run -p 5000:5000 ai-phishing-detector

REST API ENDPOINTS

Exposed endpoint for third-party system integrations:

POST /scan

// Request Payload
{
  "url": "http://paypal-verification-alert.net/login"
}

// Response
{
  "status": "success",
  "url": "http://paypal-verification-alert.net/login",
  "verdict": "Phishing",
  "confidence": 0.94,
  "risk_score": 96

CLOUD DEPLOYMENT

For persistent deployments (Render, Railway, AWS, Heroku):

  • Persistent Volume: Mount a volume to the /app/database directory so your SQLite database (user accounts and scan logs) isn't wiped on redeploys.
  • GitHub Actions: The workflow at .github/workflows/deploy.yml automatically executes tests on push.
